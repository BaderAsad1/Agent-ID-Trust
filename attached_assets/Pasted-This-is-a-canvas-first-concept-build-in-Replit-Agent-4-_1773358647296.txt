THE HERO MOMENT — THIS IS THE ENTIRE POINT OF BOTH PAGES
════════════════════════════════════════════════════════════════

The hero of both pages is ONE thing:
an animated Agent ID profile card that builds itself from nothing, live on screen, as if an agent is registering in real time.

This is the signature visual.
This is the emotional centerpiece.
This is the thing that makes the site feel game changing.

The sequence plays automatically on load, takes about 4 seconds total, then the card sits glowing and alive.
It completes once and holds.
It must feel smooth, polished, premium, and cinematic.
No jank.
No awkward pauses.
No broken staggering.
No clumsy animation timing.
If this does not feel elite, fix it before focusing on anything else.

STEP 0 (0ms):
Card shell appears — background, border, shadow.
Everything inside is invisible.
A terminal appears ABOVE the card:
$ agentid register --handle research-agent

STEP 1 (600ms):
Avatar fades in and scales up from 0.6 to 1.0 with a spring overshoot using:
cubic-bezier(0.34,1.56,0.64,1)
Agent name “Research Agent” slides up from 6px, opacity 0 to 1.

STEP 2 (900ms):
Handle “agent.id / research-agent” fades in.
Second terminal line appears:
› Claiming research.agent... checking availability ✓

STEP 3 (1300ms):
Domain badge “◈ research.agent” slides in from left:
translateX(-10px) → 0
It glows cyan.
“Active” status pill fades in top right with pulsing green dot.

STEP 4 (1700ms):
Trust score ring animates.
SVG circle stroke fills from 0% to 94% over 1.4s with ease-out.
Number counts up from 0 to 94 inside the ring using requestAnimationFrame.
Third terminal line appears:
› Identity verified. trust_score: 94  status: active ✓

STEP 5 (2200ms):
Description text fades up.
Then 4 capability chips drop in one by one with 80ms stagger:
- Research
- Web Search
- Data Analysis
- Content Creation

STEP 6 (2900ms):
Card footer slides up:
“Listed on Marketplace · $25/task” in purple
and “★ 4.9 (47)” in amber

STEP 7 (3300ms):
Card gets a subtle breathing glow.
Box-shadow pulses softly on a 3s loop.
Top edge of the card gets a 1px horizontal blue gradient line that fades in.
Cursor blinks at the end of terminal.
Card is now fully alive.

This must feel like a living internet-native identity object, not just another dashboard card.

ANIMATION SPECS — BOTH VERSIONS
════════════════════════════════════════════════════════════════

Section scroll reveals:
- use IntersectionObserver
- threshold 0.15
- opacity 0 → 1
- translateY(20px) → 0
- 600ms ease-out
- child elements staggered 80ms each

Use this keyframe:
@keyframes fadeUp {
  from { opacity:0; transform:translateY(18px); }
  to   { opacity:1; transform:translateY(0); }
}

Card build:
- use setTimeout chains and state sequencing
- not pure CSS animation-delay for the whole sequence

Spring on avatar:
- cubic-bezier(0.34,1.56,0.64,1)

Trust fill:
- stroke-dashoffset 138 → 14
- transition 1.4s cubic-bezier(0.4,0,0.2,1)

Trust counter:
- requestAnimationFrame
- easeOut over 1400ms

Chip stagger:
- transition-delay 0, 80ms, 160ms, 240ms

Button shimmer:
- ::before pseudo-element
- left -100% → 150% on hover
- 440ms

Marquee:
- duplicate items for seamless loop
- pause on hover

Breathing glow:
- CSS animation alternates base shadow with added blue glow
- 3s ease-in-out infinite

Cursor blink at end of terminal:
- subtle
- not distracting

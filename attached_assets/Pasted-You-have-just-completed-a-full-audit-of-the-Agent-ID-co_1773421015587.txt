MODERATE FIXES
M1 — Landing page to app design discontinuity
The cinematic landing page transitions jarringly into a standard dark SaaS layout on all other pages. Add a transition that softens this handoff. At minimum: ensure the nav bar that appears on interior pages uses the same font, color tokens, and spacing language as the landing page. The goal is continuity, not matching — they can look different but they should feel like the same product.
M2 — No marketplace listing creation UI
The backend supports POST /marketplace/listings but there is no frontend form. Build a marketplace listing creation form in the dashboard. Fields required: listing title, description, capability tags, pricing model (per task / per hour / fixed), price, and availability toggle. Match the existing dashboard form patterns exactly.
M3 — N+1 patterns in billing and jobs
In billing.ts (handleInvoicePaid, enforceAgentLimitsForUser) and jobs.ts (expireJobs), replace sequential for...of loops with individual DB updates with batch operations. Use Drizzle's batch update patterns or whereIn clauses to eliminate the N+1 behavior.
M4 — Missing database indexes
Add the following indexes:

users.email — standard index
users.username — standard index
agents.handle — functional index on lower(handle) to support case-insensitive lookups without full table scans
Any marketplace listing columns used in filter + sort queries — identify these from the query patterns in the routes and add composite indexes accordingly

M5 — "View API Docs" and "OpenAPI Spec" buttons on /for-agents are dead
Wire these buttons up. If a Swagger UI or hosted OpenAPI spec page exists, link to it. If it does not exist yet, generate one: serve the existing OpenAPI 3.1 spec via Swagger UI at /api/docs and link both buttons there. This is the single most important page for developer adoption.
M6 — Redis not configured
Add REDIS_URL to .env.example with a clear description. In the server startup log, upgrade the current warning to a prominent startup warning that clearly states which features are disabled without Redis. Document in a README or inline comment that domain provisioning and webhook delivery require Redis to be configured.
M7 — HMAC secret ephemeral
Add ACTIVITY_HMAC_SECRET to .env.example with a clear description and a note that without it, activity log signatures will not survive server restarts. In the server startup log, make the current warning impossible to miss.
M8 — Marketplace payments are simulated
This requires a decision. There are two valid paths:
Option A — Wire up real Stripe payment intents for marketplace transactions, replacing the pi_sim_... simulation.
Option B — Keep simulation but make it visually explicit in the UI with a "Payments in sandbox mode" badge so users and testers know.
Pick Option B for now and implement the sandbox badge. Flag Option A as a gap to wire up before public launch. Do not silently simulate real money movement.
